<feature>
  <name>Groq API key validation</name>
  <files>src/linux_speech_flow/groq_client.py, tests/test_groq_client.py</files>
  <behavior>
    **validate_api_key(api_key: str) -> dict:**

    Returns `{"ok": True}` or `{"ok": False, "message": str}`

    **Pass 1 — Format check (no HTTP):**
    - Key does not start with "gsk_" → `{"ok": False, "message": "Invalid API key format"}`
    - Key shorter than 20 chars → `{"ok": False, "message": "Invalid API key format"}`
    - If format passes → proceed to pass 2

    **Pass 2 — Live HTTP call:**
    - `GET https://api.groq.com/openai/v1/models` with `Authorization: Bearer {api_key}`, timeout=10
    - Response 200 → `{"ok": True}`
    - Response 401 → `{"ok": False, "message": "Invalid API key"}`
    - Any other status → `{"ok": False, "message": f"Unexpected response ({status_code})"}`
    - `requests.exceptions.ConnectionError` → `{"ok": False, "message": "Could not connect to Groq — check your internet connection"}`
    - `requests.exceptions.Timeout` → `{"ok": False, "message": "Could not connect to Groq — check your internet connection"}`

    **Test cases (all HTTP mocked):**
    - Empty string → format fail, no HTTP call made
    - "not_groq_key" (no gsk_ prefix) → format fail, no HTTP call made
    - "gsk_" + "x"*5 (too short) → format fail
    - "gsk_" + "a"*50 (valid format) + mock 200 → ok=True
    - "gsk_" + "a"*50 + mock 401 → ok=False, message="Invalid API key"
    - "gsk_" + "a"*50 + mock 500 → ok=False, message contains "500"
    - "gsk_" + "a"*50 + mock ConnectionError → ok=False, message contains "connect"
    - "gsk_" + "a"*50 + mock Timeout → ok=False, message contains "connect"
    - Format fail case: assert requests.get NOT called (verify no HTTP on format failure)
  </behavior>
