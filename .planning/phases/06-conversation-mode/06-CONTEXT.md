# Phase 6: Conversation Mode - Context

**Gathered:** 2026-02-21
**Status:** Ready for planning

<domain>
## Phase Boundary

Long-form recording mode (distinct from push-to-talk) that captures multi-hour sessions, silence-chunks them into auto-transcribed pieces, drives an iterative multi-model AI analysis + Q&A loop, and produces a coalesced markdown output file. This phase adds a new recording mode, a post-stop analysis dialog, a Q&A window, and a dedicated conversation file viewer. Regular push-to-talk transcription (Phase 2–3) is unchanged.

</domain>

<decisions>
## Implementation Decisions

### In-session feedback
- Feedback level is user-configurable in Settings: two modes — tray-only and status window
- Fn+D hotkey toggles between tray-only and status window during an active recording (Phase 7 will make the key itself configurable; Fn+D is the Phase 6 default)
- Status window shows: elapsed timer + chunk count + last chunk status (e.g., "Recording: 4m32s | 3 chunks transcribed | last: 45s ago")
- Transcription is silent background — no live transcript shown during recording
- 180s silence: GTK modal dialog prompts user "continue or stop?"
- 300s silence: auto-stop with audio cue (per roadmap)
- 4hr hard limit with warning sound (per roadmap); all timers configurable in Settings
- Tray icon: same animation as regular recording + a badge/indicator overlay (e.g., small "C" dot) to visually distinguish conversation mode

### Post-stop dialog
- GTK dialog appears immediately when the user stops conversation recording
- Dialog layout (top to bottom):
  1. Qualifying questions (inline, above prompt textarea) — configurable list in Settings; Claude designs the default set covering audience, complexity, and purpose
  2. Prompt textarea — pre-filled from a "default conversation prompt" setting; editable per-session
  3. Save to file checkbox (checked by default)
  4. Inject to active window checkbox
  5. AI model checkboxes: Groq, Grok, Gemini (only configured models enabled; unconfigured models grayed out with tooltip "Add API key in Settings to enable")
  6. Submit button
- API keys for Grok and Gemini configured in Settings window alongside the existing Groq key (new section or tab)
- When multiple models are selected, a configurable meta-model (default: Groq) makes a synthesis call to merge all model responses into one unified output — no side-by-side comparison, one coalesced result
- Clicking Submit opens a new Q&A window; dialog closes

### Save timing and file lifecycle
- Raw transcript saved immediately when recording stops: `ISO8601_untitled.txt` in user-configured save location
- Save location: user-configurable in Settings (default: `~/Documents/conversations/`)
- When AI analysis completes: file updated in-place AND renamed to `ISO8601_ai-title.txt` (where AI-title is generated by the analysis)
- Final file structure (top to bottom):
  ```
  [Metadata: date/time, duration, chunk count, models used]

  ## Summary
  [AI-generated summary from synthesis model]

  ## Q&A
  [Each Q&A round: AI question + user answer]

  ## Transcript
  [Full raw transcript as continuous text — no chunk markers]
  ```
- Sections use markdown headers for readability and parseability

### Q&A interaction pattern
- Q&A starts only if AI confidence is below 95% after initial analysis; if already confident, Q&A is skipped entirely
- Chat-style window: scrollable log of all previous Q&A rounds at top; current AI question + text input + speak button below
- User can rephrase the AI question before answering (editable question field)
- Speak button: hold-to-record (push-to-talk, same pattern as regular recording); spoken answer transcribed and placed in the text field
- "Done, use current output" button always visible; clicking it when AI is below 95% confidence shows a warning ("AI confidence at X% — output may be incomplete") before user confirms
- Max Q&A iterations: default 3 rounds; after 3, user is asked "continue with more rounds?" — not a hard stop
- AI confidence: AI signals its confidence level; user must also confirm before the loop ends (both conditions required)
- After the loop ends, file is updated in-place with the full coalesced content

### Conversation file viewer
- Dedicated viewer separate from the Phase 5 history viewer
- List panel (left): conversation files with metadata (date, duration, AI title)
- Preview pane (right): full coalesced content of the selected conversation
- "Continue Q&A" button per conversation entry — re-opens the Q&A window for that file

### Settings additions (Phase 6)
- Feedback level toggle (tray-only / status window)
- Default conversation prompt textarea
- Qualifying questions list (add/modify/remove individual questions)
- Meta-model for synthesis (default: Groq)
- Conversation files save location (default: ~/Documents/conversations/)
- Max Q&A iterations (default: 3)
- Auto-analyze setting (pre-checks AI analysis in post-stop dialog by default)
- API keys: Grok and Gemini (alongside existing Groq key)

### Claude's Discretion
- Specific qualifying questions to include in the default configurable list (design around audience, complexity, and purpose context)
- Badge/indicator overlay design for tray icon during conversation mode
- Exact wording and options in the GTK silence modal (180s prompt)
- Whether spoken Q&A answer auto-submits after transcription or fills the text field for review before submit (prefer review step for consistency with existing UX)
- Conversation viewer layout and sizing defaults

</decisions>

<specifics>
## Specific Ideas

- Feedback mode cycle (Fn+D) is a toggle: tray-only ↔ status window (quiet mode = tray-only, so effectively two modes)
- The qualifying questions are meant to help AI models calibrate response complexity — not everything should be treated as an enterprise deliverable
- The qualifying questions list in Settings can be extended, modified, or trimmed by the user
- Meta-model synthesis: one additional API call that reads all model responses and produces a unified result
- "Continue Q&A" from the viewer re-opens analysis on an already-saved file

</specifics>
