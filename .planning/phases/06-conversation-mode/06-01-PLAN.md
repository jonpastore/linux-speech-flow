**Note on hotkey key names (Fn key Linux limitation):**
CONTEXT.md specifies "Fn+D" as the Phase 6 default for feedback mode toggle. However, RESEARCH.md Open Question 1 explains that Fn key combos (Fn+C, Fn+D, etc.) do not produce standard pynput Key values on Linux — they are hardware-firmware intercepted and never reach the OS input layer. Therefore:
- `conv_hotkey_start` defaults to `"f11"` (Linux-practical substitute for Fn+C)
- `conv_hotkey_feedback` defaults to `"f12"` (Linux-practical substitute for Fn+D)
Phase 7 will make both keys user-configurable. The config key names preserve the user's intent; only the default values are adjusted for Linux hardware reality.

   Note: Do NOT modify or touch any existing F9/ESC/F10 branches. The IDLE guard means F11 during active regular recording is silently ignored (self._state == _STATE_RECORDING, not _STATE_IDLE).

4. Add three new GTK-thread methods:
   ```python
   def _conv_start(self) -> bool:
       if self._state != self._STATE_IDLE:
           return False
       self._state = self._STATE_CONVERSATION
       if self._on_conv_start_cb:
           self._on_conv_start_cb()
       return False

   def _conv_stop(self) -> bool:
       if self._state != self._STATE_CONVERSATION:
           return False
       self._state = self._STATE_IDLE
       if self._on_conv_stop_cb:
           self._on_conv_stop_cb()
       return False

   def _conv_feedback_toggle(self) -> bool:
       if self._on_conv_feedback_cb:
           self._on_conv_feedback_cb()
       return False
   ```
