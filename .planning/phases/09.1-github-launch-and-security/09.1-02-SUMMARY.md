## Decisions Made

- `os.chmod` placed after `with self._connect() as conn:` block exits — ensures connection is fully closed before chmod to avoid any ordering issues with SQLite file operations
- Slack wizard page is informational only (no token entry) — Slack OAuth app creation is multi-step and better served by README + Settings > Integrations

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed Wave 0 test stubs in test_onboarding_dialog.py**
- **Found during:** Task 2 (OnboardingDialog implementation)
- **Issue:** Wave 0 stubs used `gi_patch` context manager with `OnboardingDialog(...)` instantiation. When `Gtk.ApplicationWindow` is a MagicMock, MagicMock's `__call__` intercepts instantiation, returning a MagicMock instead of a real Python object. MagicMock's `__setattr__` stores callback references in the mock's internal structure, not in instance `__dict__`. As a result, `self._on_continue` reads back as a new MagicMock child rather than the passed callback — making `on_continue.assert_called_once()` always fail.
- **Fix:** Rewrote `test_on_continue_fires` and `test_on_quit_fires` to import `OnboardingDialog` outside `gi_patch` (using the real class), then use `OnboardingDialog.__new__(OnboardingDialog)` to skip GTK `__init__`, and assign callbacks directly on the GObject instance. `test_construction` retains the `gi_patch` pattern (it only checks `dlg is not None`, which works).
- **Files modified:** `tests/test_onboarding_dialog.py`
- **Verification:** All 3 `TestOnboardingDialog` tests GREEN; 341 total tests pass
- **Committed in:** `aea8af1` (Task 2 commit)
