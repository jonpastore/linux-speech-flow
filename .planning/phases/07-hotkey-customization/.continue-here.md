## Bug fixes delivered this session (not Phase 7 proper):

- **Window capture race condition**: `get_active_window_info()` moved from
  `_on_conv_session_complete` to `_on_conv_start` (app.py) so the target app
  (gedit, etc.) is captured before our dialogs appear.

- **Target window display**: Both `ConversationDialog` and `TranscriptOutputWindow`
  now show "Target: <window name>" label. Paste button in TranscriptOutputWindow
  reads "Paste to <window name>".

- **Save to file**: Now opens `Gtk.FileChooserNative` (browse + rename) instead
  of auto-saving. Response handler shows "Saved: /path" or "Save failed: reason".
  `coalesce_file` now uses `encoding="utf-8"`.

- **Silence dialog dismissed by speech**: `_silence_dialog.close()` moved outside
  `if self._stop_timer:` block in `_reset_silence_timers` (conversation_manager.py)
  so speech always dismisses it regardless of timer state.

- **First recording dropped**: Auto-calibration in conversation_recorder.py now
  skips calibration when `ambient >= current_threshold` (user was speaking during
  guard period, inflating the ambient sample → false high threshold → chunk discarded).

- **Qualifying question checkboxes**: ConversationDialog now shows a CheckButton
  per question (checked by default). Unchecking greys out the answer entry and
  excludes the Q&A from the AI prompt.

- **Hotkey rewrite** (Phase 7 start): hotkey.py fully rewritten:
  - Ctrl+Alt+R → start/stop recording
  - Ctrl+Alt+C → start/stop conversation
  - Ctrl+Alt+P → reprocess failed
  - Ctrl+Alt+F → toggle feedback mode
  - ESC → stop recording
  - Added `_ctrl_held`/`_alt_held` modifier tracking via `_on_release`
  - Added static `_key_letter(key)` helper (checks vk 65-122 then key.char)
  - Renamed `_stop_was_f9` → `_stop_was_hotkey`

- **Tests**: 129/129 passing at end of session (hotkey tests NOT yet updated —
  they reference old F-key API and will now FAIL).

<decisions_made>

- Ctrl+Alt combinations chosen (user confirmed) over F-keys to avoid conflicts with
  gedit/terminal fullscreen (F11), bookmarks (F9), etc.
- Physical Fn key is not interceptable on Linux — documented and explained to user.
- `_key_letter()` static method checks `vk` (97-122 a-z, 65-90 A-Z) then falls back
  to `key.char.lower()` — covers X11 keysym and pynput char modes.
- `_stop_was_f9` renamed to `_stop_was_hotkey` for accuracy.
- Calibration skip: `if ambient >= self._silence_rms_threshold` — if guard frames
  contain audio-level signal, user was speaking; keep sensitive default threshold.
- Requirements audit scope: full (roadmap + PROJECT.md + per-phase CONTEXT files).

</decisions_made>
