  <action>
Extend ConversationRecorder to support an optional silence tick callback.

Changes in __init__:
- Add `self._on_silence_tick = None` after `self._on_error = None`

Changes in start():
- Add `on_silence_tick=None` as optional third parameter after `on_error`
- Store: `self._on_silence_tick = on_silence_tick`
- Update the docstring to document on_silence_tick: `Callable(silence_frames: int) — GTK main thread via GLib.idle_add; silence_frames is the current consecutive silent frame count within the current chunk (0 = voice detected); emitted only when value changes (debounced).`

Changes in _record_one_chunk():
- Add `last_emitted_silence = -1` immediately after `had_audio = False` (before the wave.open block)
- After the `if rms < SILENCE_RMS_THRESHOLD:` / `else:` block (and before the `if silence_frames >= silence_limit_frames:` check), add:

```python
                    # Emit silence tick only when value changes (debounced to avoid GTK queue pressure)
                    if self._on_silence_tick and silence_frames != last_emitted_silence:
                        last_emitted_silence = silence_frames
                        GLib.idle_add(self._on_silence_tick, silence_frames)
```

Do NOT emit tick during the guard frames window (frames_written < MIN_GUARD_FRAMES) — the existing `if frames_written >= MIN_GUARD_FRAMES:` block already gates the RMS check, so the new emit line goes inside that same block, after the silence/voice branch.

Do NOT call GTK methods directly from this thread — only GLib.idle_add is safe here.

The callback signature is `on_silence_tick(silence_frames: int) -> bool` where the return value must be False (GLib.idle_add contract). The recorder emits the raw frame count; the manager converts to seconds.
