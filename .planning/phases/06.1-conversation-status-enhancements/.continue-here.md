<remaining_work>

**BUG 1 — Mic level DrawingArea doesn't animate (HIGH PRIORITY)**
- User reports: "slider still doesn't slide" in ConversationStatusWindow
- `_draw_mic` on `_mic_canvas` uses `queue_draw()` triggered by `update_mic_level()`
- `on_silence_tick` DOES work (silence counter animates), so GLib.idle_add from recorder thread works
- Root cause UNKNOWN — likely `queue_draw()` not triggering GTK repaints OR `on_audio_level` callback has a silent failure
- Next step to try: Replace `queue_draw()` triggered by update with a `GLib.timeout_add(50, ...)` periodic redraw timer started in `start()` and stopped in `stop()`. This decouples redraw from callback.
- Also: User wants a SEPARATE Gtk.Scale slider BELOW the DrawingArea in ConversationStatusWindow (same layout as Settings) so they can adjust threshold during a session (not just in Settings)

**BUG 2 — Auto-stop (force end) doesn't work (HIGH PRIORITY)**
- User set warn=10s, stop=20s; 30s prompt fired but force-end did NOT happen
- Root cause FOUND: `_on_silence_warn()` computes `extra = max(stop_sec - warn_sec, 30)` — the `max(..., 30)` clamp means minimum extra time is 30s regardless of setting! With 10/20, extra=max(10,30)=30, so auto-stop fires at 10+30=40s, not 20s
- Fix: change `max(..., 30)` to `max(..., 5)` in `conversation_manager.py:_on_silence_warn()`
- Also: the `_show_silence_modal()` creates a dialog that STAYS OPEN — the auto-stop timer fires anyway but `stop_session()` may conflict with dialog state. Need to close dialog before stopping.

<decisions_made>

- Replaced `Gtk.LevelBar + Gtk.Overlay` with a single `Gtk.DrawingArea` for mic level — avoids GTK4 sizing/compositing issues. Level fill: blue when below threshold, orange when above. White vertical threshold marker line.
- `RMS_DISPLAY_SCALE = 12` — multiplier so typical speech reaches ~0.5–1.0 on the 0–1 display scale.
- Config migration strategy: only migrate values that are EXACTLY at old defaults (180/300→30/60). Users who deliberately set 180 or 300 would also get migrated (acceptable tradeoff).
- `on_audio_level` fires every frame (100ms) — intentionally high frequency for smooth level display. `on_silence_tick` fires only on value change (debounced).
- `conv_silence_rms_threshold` range in settings slider: 0.001–0.05, step 0.001.
- Silence timeout settings are in Conversation section (not Audio section).
</decisions_made>
