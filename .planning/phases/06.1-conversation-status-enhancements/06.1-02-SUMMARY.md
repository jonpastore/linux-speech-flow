## Decisions Made

- **Cross-chunk silence tracking:** The recorder's `silence_frames` is per-chunk and resets to 0 on each new chunk. When `_on_silence_tick` receives a value less than the previous value, it signals a new chunk started after a silence boundary — the previous chunk's max silence is added to `_silence_offset_sec`. Display = offset + current frames.
- **Voice reset:** When `silence_frames == 0` (voice detected), both `_silence_offset_sec` and `_last_silence_frames` reset to 0, ensuring the display cleanly shows 0 when speech resumes.
- **GTK card styling:** Used the built-in `card` CSS class rather than custom CSS — it's maintained by the GNOME/libadwaita design system and provides consistent appearance across themes.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Silence counter capped at ~3 seconds due to per-chunk reset**
- **Found during:** Human verification (Task 2 checkpoint)
- **Issue:** The recorder splits chunks at the silence threshold (3s). Each new chunk resets `silence_frames` to 0, causing the status window to show "Silence: 3s" then immediately "Silence: 0s" and re-climb — effectively capping visible silence at the chunk split interval.
- **Fix:** Added `_silence_offset_sec` and `_last_silence_frames` to `ConversationManager`. When `_on_silence_tick` fires with a value smaller than the previous (chunk boundary detected), the prior chunk's silence is accumulated into the offset. Display shows `offset + current_frames_in_seconds`.
- **Files modified:** `src/linux_speech_flow/conversation_manager.py`
- **Verification:** Logic trace-verified; unit-testable without audio hardware
- **Committed in:** `89145f4`

**2. [Rule 2 - UI Enhancement] Transcript label rendered as plain unstyled text**
- **Found during:** Human verification (Task 2 checkpoint)
- **Issue:** The transcript label appeared as plain text with no visual container, making it look unprofessional and hard to distinguish from the status line above it.
- **Fix:** Wrapped in `Gtk.Frame` with `card` CSS class and inner `Gtk.Box` providing 10px horizontal / 8px vertical margins. Applied `body` CSS class to label for native GNOME typography.
- **Files modified:** `src/linux_speech_flow/conversation_status.py`
- **Verification:** Confirmed renders with visual card container in GTK 4 applications
- **Committed in:** `ec3cd9c`
